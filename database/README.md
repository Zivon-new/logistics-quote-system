# 数据库字段说明

## 📊 数据库表结构

### 1. routes (路线表)
**主表，存储路线基本信息**

| 字段名 | 类型 | 说明 | 必填 | 默认值 |
|--------|------|------|------|--------|
| 路线ID | int | 主键，自增 | 是 | AUTO |
| 起始地 | varchar(100) | 起始地名 | 是 | - |
| 途径地 | varchar(100) | 途径地名 | 否 | NULL |
| 目的地 | varchar(100) | 目的地名 | 是 | - |
| 交易开始日期 | date | 交易周期开始日期 | 否 | NULL |
| 交易结束日期 | date | 交易周期结束日期 | 否 | NULL |
| 交易年份 | year | 虚拟列，自动计算 | - | - |
| 交易月份 | tinyint | 虚拟列，自动计算 | - | - |
| 实际重量(/kg) | decimal(18,2) | 路线总实际重量（千克） | 否 | 0.00 |
| 计费重量(/kg) | decimal(18,2) | 路线计费重量（千克） | 否 | NULL |
| 总体积(/cbm) | decimal(18,3) | 路线总体积（立方米） | 否 | NULL |
| 货值 | decimal(18,2) | 货值 | 否 | 0.00 |
| 创建时间 | datetime | 创建时间 | - | CURRENT_TIMESTAMP |

**JSON输出字段映射**:
```json
{
  "路线ID": 1,
  "起始地": "深圳",
  "途径地": null,
  "目的地": "新加坡",
  "交易开始日期": "2025-10-20",
  "交易结束日期": "2025-10-24",
  "实际重量(/kg)": 100.00,
  "计费重量(/kg)": 100.00,
  "总体积(/cbm)": 5.000,
  "货值": 50000.00
}
```

## ⚠️ 重要说明

### 自动计算字段
以下字段由触发器自动计算，不需要在JSON中提供：
- `routes.交易年份` - 从交易开始日期自动计算
- `routes.交易月份` - 从交易开始日期自动计算
- `fee_items.人民币金额` - 从原币金额和汇率自动计算
- `fee_total.人民币金额` - 从原币金额和汇率自动计算
- `goods_details.总重量(/kg)` - 从数量×重量自动计算
- `goods_details.总价` - 从数量×单价自动计算

### 默认值
- 币种字段默认为 'RMB'
- 数值字段默认为 0 或 0.00
- 创建时间自动设置为当前时间

### 外键约束
- 删除路线时，会级联删除相关的代理路线、货物等
- 删除代理路线时，会级联删除相关的费用、汇总等
